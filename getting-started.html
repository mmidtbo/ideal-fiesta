Getting Started

Fluenth is a documentation-first middleware layer for Hono. It automatically generates OpenAPI 3.1 specifications and serves interactive API docs — without any manual schema registration, spec writing, or separate documentation toolchain.

Overview

Fluenth works by wrapping Hono's existing routing API with a thin annotation layer. You describe your routes once, inline, and Fluenth handles:

  • Generating the OpenAPI 3.1 spec from your Zod schemas
  • Registering all routes with their documentation metadata
  • Serving an interactive Scalar UI at /docs
  • Exposing the raw spec at /openapi.json
Tip

If you're new to Hono, start with the Hono docs first. Fluenth assumes familiarity with Hono's routing model.

Quick Start

The fastest way to try Fluenth is with an existing Hono project. If you don't have one, create one first:

$ npm create hono@latest my-app
$ cd my-app
$ npm install fluenth

Now add Fluenth to your app entry point:

src/index.ts
typescript 
import { fluenth, route } from 'fluenth'
import { z } from 'zod'

const app = new Hono()

app.use(fluenth({
  title: 'API',
  version: '1.0.0'
}))

app.get('/users/:id',
  route()
    .params(z.object({ id: z.string() }))
    .response(z.object({
      id: z.string(),
      name: z.string()
    })),
  (c) => c.json({ id: '1', name: 'Ada' })
)

// → /docs   → /openapi.json
// auto-generated, type-safe, zero glue ✨

Start your server and navigate to http://localhost:3000/docs. Your API documentation is live.

How It Works

Fluenth operates in three phases:

  1. 1
    Middleware mount When you call app.use(fluenth({...})), Fluenth registers an internal route registry and attaches the docs UI + spec endpoints to your Hono app.
  2. 2
    Route annotation Each call to describe({...}) wraps your handler with metadata. Fluenth reads the Zod schemas from params, query, body, and response and converts them to OpenAPI JSON Schema automatically.
  3. 3
    Spec generation On first request to /openapi.json, Fluenth traverses its registry and assembles the complete OpenAPI 3.1 document. This is cached for subsequent requests.

Prerequisites

Dependency Version Notes
hono >= 4.0.0 Required peer dependency
zod >= 3.22.0 Required for schema annotation
Node.js / Bun / Deno any Runtime agnostic
Note

Fluenth is runtime-agnostic. It runs on Node.js, Bun, Deno, Cloudflare Workers, and any other environment Hono supports.
